# DataChat — Architecture

> An AI-powered data-analysis notebook. Users converse with a tabular dataset
> in English; the LLM emits Python; the server executes it inside a sealed
> sandbox; results stream back as Plotly figures, scalars, or tables.

## Goals & non-goals

**Goals**
- End-to-end answer in **<3 s p50, <6 s p95** for typical questions on
  10k-row datasets.
- Code generated by the LLM is executed in a process whose blast radius is
  bounded by the kernel, *not* by the LLM's good intentions.
- Useful *without* an OpenAI key — a deterministic `MockLLMProvider` serves
  the same SSE protocol so the UI, sandbox, and tests run hermetically.
- Single-process FastAPI binary; SQLite by default, Postgres swappable.

**Non-goals**
- Multi-tenant isolation. The sandbox protects the *server* from runaway or
  malicious code, not multiple paying customers from each other. Use a
  container or VM for that.
- Browser-side XSS hardening of arbitrary Plotly JSON — we trust our own
  Plotly figure shape and let `react-plotly.js` render it. If the figure
  source ever becomes user-controlled HTML, audit must precede that change.
- Full RAG / embedding pipeline. The dataset is loaded as `df`; the model
  reasons over its head/sample, not vector chunks.

## Streaming protocol (SSE)

`POST /v1/sessions/{id}/messages` returns `text/event-stream`.
Frames in the order they arrive:

| event           | payload                                                | when |
|-----------------|--------------------------------------------------------|------|
| `user_message`  | `{id, content}`                                        | once, immediately after persist |
| `token`         | `{delta}`                                              | repeated, one per chunk from the LLM |
| `finish`        | `{reason}`                                             | when the LLM stream ends |
| `code`          | `{code}`                                               | once, after extracting the first ```python``` block |
| `exec_result`   | `{ok, figure, result_repr, stdout, stderr, elapsed_ms, error_class, error_message}` | once, after sandbox returns |
| `error`         | `{error}`                                              | only on LLM-side failure |
| `done`          | `{message_id, elapsed_ms}`                             | always last |

SSE was chosen over WebSocket because:
- One-way streaming fits the protocol exactly.
- Survives any HTTP/1.1 reverse proxy, CDN, or WAF that mangles WebSocket.
- `Cache-Control: no-cache` + `X-Accel-Buffering: no` are sufficient to
  defeat all the proxies we tested in dev.

## Sandbox model

The threat is **untrusted Python from an LLM**. Defenses, in depth:

1. **Separate process.** Every run forks a fresh `python` subprocess. No
   shared interpreter state with FastAPI; no monkey-patched stdlib leaking
   between turns.
2. **POSIX RLIMITs** applied via `preexec_fn` *before* `exec`:
   - `RLIMIT_CPU` — wall-cpu seconds (default 8 s)
   - `RLIMIT_AS`  — virtual address space (default 512 MiB)
   - `RLIMIT_FSIZE` — max single-file write size
   - `RLIMIT_NPROC` — process count cap (defends against fork bombs)
3. **`sys.addaudithook` (PEP 578)** installed inside the harness:
   - Forbidden modules at import time:
     `socket, ssl, subprocess, os, shutil, ctypes, multiprocessing,
     asyncio, threading, urllib*, http*, httpx, requests, ftplib, smtplib,
     telnetlib, poplib, pickle, shelve, marshal`.
   - Forbidden audit events:
     `subprocess.Popen, os.system, os.exec*, os.spawn*, os.fork, os.forkpty,
     socket.connect, socket.bind`.
4. **Minimal env.** `subprocess.exec` is invoked with an env dict containing
   only `PATH`, `LANG`, `PYTHONIOENCODING`, `PYTHONHASHSEED`. No secrets
   propagate into the child.
5. **Confined cwd.** Each run gets a fresh `TemporaryDirectory`. The dataset
   is **copied** (not symlinked) into that dir so the child cannot follow a
   symlink to escape. The harness only ever opens `./<dataset>.csv`.
6. **Wall-clock kill.** A parent-side `asyncio.wait_for` enforces a wall
   timeout independent of `RLIMIT_CPU`, defending against I/O-blocked
   processes that never burn CPU.

The audit hook is *not* a security boundary by itself — a determined
attacker can disable it with low-level tricks. Layers 1, 2, and 4 are the
real walls. The audit hook exists to catch the obvious 99% case early
with a clean error message instead of a SIGKILL'd process.

## Data model (`SQLAlchemy 2`)

```
sessions(id PK, dataset, title, created_at)
messages(id PK, session_id FK, role, content, code, figure_json, elapsed_ms, status, created_at)
runs(id PK, message_id FK, exit_code, elapsed_ms, peak_mem_bytes, stdout, stderr, error_class)
```

- `sessions.created_at` indexed for the recent-session list.
- `messages.session_id` indexed for the detail view.
- `figure_json` stored as TEXT (Plotly JSON is small; ~3-30 KB typical).
- No retention policy in v0.1; ops can `DELETE WHERE created_at <` cron'd.

## LLM provider

```
class LLMProvider(Protocol):
    async def stream(self, messages, *, model: str) -> AsyncIterator[Token]:
        ...
```

Two implementations:
- `OpenAIProvider` — wraps `openai.AsyncClient` with the streaming chat API.
  Default model `gpt-4o-mini`. Token budget capped at 4 096 to keep the
  sandbox response bounded.
- `MockLLMProvider` — pattern-matches the prompt against six question
  archetypes (row count, top-N, distribution, by-month, summary, head)
  and emits a deterministic Python program. Streams in 16-char chunks at
  `chunk_delay_ms=12` so the UI animation looks identical to the real
  thing in tests, demos, and CI.

The provider is injected into `build_app(...)`. Tests pass `MockLLMProvider`
directly; production reads `DATACHAT_LLM_PROVIDER=openai|mock`.

## Frontend

- Vite + React 18 + TypeScript strict.
- `lib/api.ts` is a hand-rolled SSE consumer using `fetch` + `getReader()`
  — `EventSource` doesn't allow POST bodies, and we want to ship the
  message in the request, not the URL.
- `react-plotly.js` is dynamically imported in `FigurePlot.tsx` so the
  initial bundle is small and the empty-state typography paints
  immediately.
- The aesthetic is **editorial-research-paper**: Bodoni Moda display
  italic, IBM Plex Sans body, JetBrains Mono code, burnt sienna `#d76040`
  on `#0c0c10` ground. Each chat turn is preceded by a hairline rule with
  a kicker label (`01 — INQUIRY`, `02 — ANALYSIS`) — the thread reads as
  a typeset journal, not a messaging app.

## Observability

- `/v1/stats` returns `count, p50_ms, p95_ms, p99_ms` from a rolling
  `deque(maxlen=512)` of per-message latencies.
- `/healthz` returns `{ok, datasets}`; the frontend's masthead shows live
  p50/p95 next to the volume number — useful while load-testing.
- Every sandbox run records `elapsed_ms`, `peak_mem_bytes`, `exit_code` to
  the `runs` table for offline analysis.

## Performance targets

| Stage                            | Target  | How |
|----------------------------------|---------|-----|
| Sandbox spawn → harness ready    | <80 ms  | warm `python -B`, no site packages bloat |
| LLM first token (Mock)           | <40 ms  | deterministic, in-process |
| LLM first token (OpenAI)         | ~600 ms | network bound; not on us |
| End-to-end p50 (Mock + 10k rows) | <3 s    | streaming + small Plotly figure |
| End-to-end p95                   | <6 s    | dominated by sandbox + serialize |

## Failure modes & how we surface them

| Failure | Surfaced as |
|---------|-------------|
| LLM 5xx / network error | SSE `error` frame; assistant message stored with `status=error` |
| Code block missing | `code` event with empty string; `exec_result` skipped; assistant text shown verbatim |
| Sandbox import-block hit | `exec_result.ok=false`, `error_class=AuditError`, message in `error_box` |
| RLIMIT_CPU exceeded | child SIGKILL'd; `exec_result.ok=false`, `error_class=Timeout` |
| Wall-clock exceeded | parent kills child; `exec_result.ok=false`, `error_class=WallTimeout` |
| Figure JSON malformed | client `safeParse` returns null; figure-wrap not rendered, code + repr still shown |

## Tests

- `tests/test_sandbox.py` — 11 cases: pandas happy path, dataset loaded as
  `df`, blocked socket/subprocess/os.system, CPU/memory/wall timeouts,
  Plotly figure round-trip, missing-dataset clean error, user-code
  exception class surfaced.
- `tests/test_http.py` — TestClient + SSE parser; full `user_message →
  token+ → code → exec_result → done` envelope; oversized message → 422.
- `tests/test_extract.py` — fenced-block extraction edge cases.
- `tests/test_store.py` — Session/message/run round-trip + indexing.

Benchmarks (`pytest -m benchmark`) measure sandbox cold-start, mock-LLM
throughput, and end-to-end SSE wall time.
